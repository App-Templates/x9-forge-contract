# Plan 04-04: Forge Migration — X9Client, voice.ts, factory Discovery End-to-End on createBridgeClient

## Goal

Migrate all Forge-side HTTP call sites targeting X9 to use `createBridgeClient`
with the typed endpoint methods delivered in plans 04-01 and 04-02. This
eliminates the last manual `fetch()` calls to cross-repo endpoints in the
`forge-v2` repo.

Three file clusters are migrated:

1. **`services/factory/src/services/x9.client.ts`** — the existing `X9Client`
   class already imports `createBridgeClient` (Phase 3 partial migration) but
   only `reload()` uses it. `stop()`, `listAgents()`, `discoverCapabilities()`
   (manifest fetch), `getEnvSchema()` still use raw `fetch`. Complete the
   migration so EVERY cross-repo call goes through the typed client.

2. **`services/voice/src/routes/voice.ts`** — the `/api/voice/webhook` handler
   at line 106 does a manual `fetch(${x9CapVoiceUrl}/webhook/post-call, ...)`
   to forward the post-call webhook to X9 cap-voice (with `X-Internal-Token`).
   This is the Forge side of the Bug #15 migration — switch to
   `createBridgeClient<'token'>(...).postCallWebhook(body)`.

3. **`services/factory/src/routes/health.ts`** (line 113) and
   **`services/factory/src/services/deploy.machine.ts`** (line 280) — direct
   fetch calls to `/internal/agents` and `/manifest` respectively. Route these
   through the same `X9Client` (after migration) or a typed local bridge
   client, eliminating direct URL construction.

SSE flow: `/internal/turn/stream` is consumed by cap-glasses (X9-internal), not
by Forge — so `internalTurnStream()` usage is in plan 04-03, not here.

This plan is the Forge side of Phase 4 SC-2 ("Forge `X9Client` e X9 `cap-voice
→ forge /api/voice/register` usano `createBridgeClient`"), SC-3 (contract tests
on real fixtures from staging), SC-5 (standardized response shape handling),
SC-6 (staging smoke). Plan 04-03 runs in parallel on the X9 side.

private mapError(op: string, err: unknown): never {
  if (err instanceof BridgeHttpError) {
    // Prefer standardized error shape if present
    const body = err.body as { ok?: boolean; code?: string; message?: string; error?: string } | null;
    const reason = body?.message ?? body?.error ?? `http ${err.status}`;
    const code = body?.code ?? (err.status === 401 ? 'X9_AUTH' : err.status === 404 ? 'X9_NOT_FOUND' : 'X9_HTTP');

    (wrapped as Error & { code?: string; status?: number }).code = code;

  throw err;
}
