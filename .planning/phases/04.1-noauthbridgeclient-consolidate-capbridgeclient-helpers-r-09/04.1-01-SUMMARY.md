# Plan 04.1-01 — Summary

## Outcome

Closes R-09: `createBridgeClient` factory now accepts `'none'` auth and returns
a typed `NoAuthBridgeClient` exposing the 3 capability discovery endpoints
(`capManifest`, `capEnvSchema`, `capHealth`). Forge and X9 can now drop their
local `capBridgeClient` helper duplications — Plan 04.1-02 consumes this in Forge.

## What shipped

### Source changes (task 01)

- `src/http/bridge-client.ts`
  - New `NoAuthBridgeClient extends BaseBridgeClient<'none'>` interface (3 methods, all `(signal?: AbortSignal)`)
  - Widened generic on `BridgeClientConfig<A>`, `BaseBridgeClient<A>`, `BridgeClient<A>`, and `createBridgeClient<A>` to `'secret' | 'token' | 'none'`
  - New `'none'` branch in factory body. Typed methods call `CapManifestResponseSchema.parse()` / `CapEnvSchemaResponseSchema.parse()` / `CapHealthResponseSchema.parse()` on raw responses (WR-01 runtime-parse discipline)
  - Factory uses `isSecret` / `isToken` flags for explicit branching (eliminates old implicit `else` fall-through)
- `src/http/endpoints/cap-manifest.ts` · `cap-env-schema.ts` · `cap-health.ts`
  - Path strings corrected from `/:cap/manifest` → `/manifest` (and equivalents). Capability identity is conveyed by the caller's `baseUrl` (Docker hostname), not a path prefix. Confirmed against Forge production usage (`forge-v2/services/factory/src/services/x9.client.ts:42` calls `${baseUrl}/manifest`). JSDoc and runtime path field both updated.
- 3 `tests/http/endpoints/cap-*.test.ts` path assertions updated to match the wire-correct values.

### Tests (task 02)

- New `tests/http/no-auth-client.test.ts` (11 tests):
  - 3 happy-path typed-method calls (verify parsed shape + correct wire path + GET method + no auth headers)
  - AbortSignal propagation
  - ZodError on invalid server response
  - BridgeHttpError on 500
  - No `X-Internal-*` headers leak on noauth client
  - 5 `@ts-expect-error` directives: noauth client rejects `listAgents` / `reloadAgent` / `internalTurn` / `voiceRegister` / `postCallWebhook`
  - Endpoint-contract path regression guard
- Appended 5 `@ts-expect-error` directives in `tests/http/bridge-client.test.ts`: secret/token clients do NOT gain `capManifest` / `capEnvSchema` / `capHealth` methods (guard vs widened generic leaking across the discriminated union)

### Barrel (task 03)

- `src/http/index.ts`: added `NoAuthBridgeClient` to the typed re-exports next to `SecretBridgeClient` / `TokenBridgeClient`.
