# Plan 04-03: X9 Migration — cap-voice → createBridgeClient, agent-core Internal Routes Using Bridge Schemas, cap-glasses SSE Parser

## Goal

Migrate all X9 call sites and server-side Fastify routes to consume the bridge
contracts delivered in plans 04-01 and 04-02.

Three call-site categories in the X9 repo:

1. **X9 → Forge** (outbound): `cap-voice/src/tools/voice-call.ts` currently does
   a manual `fetch(FORGE_VOICE_REGISTER_URL, ...)` to `POST /api/voice/register`.
   Replace with `createBridgeClient({ auth: { type: 'token', ... } }).voiceRegister(...)`.
2. **X9 internal server** (inbound from Forge / internal): `agent-core/src/index.ts`
   registers the 6 `/internal/*` routes (`GET /internal/agents`, `POST /internal/agents/:agentId/reload|stop`,
   `POST /internal/turn`, `POST /internal/turn/stream`, `POST /internal/query`) plus
   `cap-voice` serves `POST /webhook/post-call`. Integrate the bridge Zod request
   schemas at the Fastify route boundary so request validation uses the single
   source of truth from `@x9-forge/contracts/http`.
3. **X9 → X9 internal clients** (intra-repo): `cap-glasses/src/agent-bridge.ts` calls
   `POST /internal/turn/stream` and hand-parses SSE frames. Migrate to
   `createBridgeClient({ auth: { type: 'secret', ... } }).internalTurnStream(body)`
   from 04-02, replacing the ~60 lines of manual buffer management. `cap-voice`
   fallback calls `POST /internal/query` via `internalQuery(body)`.

The migration is backwards-compatible during deploy: bridge schemas type BOTH
the legacy response shape (`{ agents: [...] }`, `{ ok: true, agentId }`,
`{ answer }`, `{ received: true }`) AND the standardized `{ ok, data }` /
`{ ok, code, message }` shape. X9 server handlers continue to emit the legacy
shape in this plan. Full shape standardization is deferred to a later milestone
— this plan tracks the wire contract migration only, not the response body
rewrite.

This plan is the X9 side of Phase 4 SC-2 ("Forge `X9Client` e X9 `cap-voice →
forge /api/voice/register` usano `createBridgeClient`") and SC-4 ("SSE
`POST /internal/turn/stream` frame shape tipizzato"). Plan 04-04 handles the
Forge side in parallel (same wave).

async function queryAgentCore(question: string, sessionId: string | undefined): Promise<string> {
  const session = sessionId !== undefined ? sessionManager.get(sessionId) : undefined;
  const contextPart = session !== undefined && session.brief.length > 0 ? session.brief : undefined;

  // NOTE: /internal/query requires X-Internal-Secret (added Phase 3 AUTH-03).

async function* generator(): AsyncGenerator<string> {
  const client = createBridgeClient({
    baseUrl: agentCoreUrl,
    auth: { "X-Internal-Secret": internalSecret },
  });
  const body: InternalTurnRequest = {
    channelId: "glasses",
    sessionId: sanitizedSessionId,
    message: question,
    history: history ?? [],
  };

  let events: AsyncGenerator<import("@x9-forge/contracts/http").ParsedSseEvent>;
  try {
    events = await client.internalTurnStream(body, signal);
  } catch (err) {
    if (err instanceof BridgeHttpError) {
      const bodyMsg = (err.body as { error?: string; message?: string } | null);
      const msg = bodyMsg?.error ?? bodyMsg?.message ?? `agent-core SSE error ${err.status}`;
      const wrapped = new Error(msg);
      rejectDone(wrapped);
      throw wrapped;
    }
    rejectDone(err);
    throw err;
  }

  let accumulated = "";
  let finalHistory: ReadonlyArray<CapGlassesLLMMessage> = history ?? [];

  for await (const event of events) {
    if (event.kind === "heartbeat") continue;
    if (event.kind === "parse_error") {
      logger.warn({ raw: event.raw, err: event.error }, "failed to parse SSE frame");
      continue;
    }
    // event.kind === 'frame' — discriminated union on frame.type
    switch (event.frame.type) {
      case "text":
        accumulated += event.frame.delta;
        yield event.frame.delta;
        break;
      case "done":
        if (event.frame.updatedHistory) {
          finalHistory = event.frame.updatedHistory as ReadonlyArray<CapGlassesLLMMessage>;
        }
        if (event.frame.reply.length > 0) accumulated = event.frame.reply;
        resolveDone({ reply: accumulated, updatedHistory: finalHistory });
        return;
      case "error": {
        const err = new Error(`agent-core stream error: ${event.frame.message}`);
        rejectDone(err);
        throw err;
      }
      case "aborted":
        logger.info({ sessionId: sanitizedSessionId, reason: event.frame.reason }, "agent-core stream aborted");
        resolveDone({ reply: accumulated, updatedHistory: finalHistory });
        return;
      case "tool_call_start":
      case "tool_call_end":
        // No-op for the text stream consumer (parity with legacy behaviour)
        break;
    }
  }

  // Stream ended without a `done` event — settle with what we got
  resolveDone({ reply: accumulated, updatedHistory: finalHistory });
}
