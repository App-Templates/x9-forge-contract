# Plan 02-03: X9 Migration — AgentContextRuntime + parseAgentContext Integration

## Goal

X9 defines `AgentContextRuntime` locally (NOT in bridge) extending `AgentContextCore`
with Runtime-only fields. `agent-manager.ts` uses `parseAgentContext` from the bridge
to validate `context.json`, then extends the result to `AgentContextRuntime`. Compat
shim ensures existing X9 services using `AgentContext` name compile without changes.
Forge `x9.ts` replaces local `X9AgentContext` with bridge re-export. Zero breaking
runtime — all production context.json files parse successfully.

```typescript
/**
 * AgentContextRuntime — X9-local extension of the cross-repo AgentContextCore.
 *
 * These fields are derived at boot time by agent-manager.ts from either:
 * - Conventions (workspacePath, registryPath derived from agentId)
 * - Credentials (telegramBotToken extracted from credentials['TELEGRAM_BOT_TOKEN'])

 *
 * NOT exported to the bridge — X9 implementation detail only.
 *

export interface AgentContextRuntime extends AgentContextCore {
  /** Absolute path: /data/workspaces/{agentId} */

  /** Absolute path: /data/agents/{agentId}/registry.json */
  registryPath: string;
  /** Telegram bot token, extracted from credentials['TELEGRAM_BOT_TOKEN'] */
  telegramBotToken: string;
  /** Human-readable agent name for logs */
  displayName: string;
}
```

Key decisions:

- 4 Runtime-only fields: `workspacePath`, `registryPath`, `telegramBotToken`, `displayName`
- `telegramBotToken` extracted from credentials (not a separate Core field)
- This interface is NOT in the bridge (AGNT-03 requirement)

```typescript
} from '@x9-forge/contracts/agent';
```

2. Update the `load` method to use the extended schema and build Runtime:
```typescript
async load(contextPath: string): Promise<AgentContextRuntime> {
  const raw = JSON.parse(await readFile(contextPath, 'utf-8')) as unknown;
  const ctx = AgentContextSchema.parse(raw) as AgentContextRuntime;
  this.contexts.set(ctx.agentId, ctx);
  this.inFlightTurns.set(ctx.agentId, 0);
  return ctx;
}
```

3. Update Map type and method signatures:
```typescript
private contexts = new Map<string, AgentContextRuntime>();

get(agentId: string): AgentContextRuntime | undefined {
  return this.contexts.get(agentId);
}

getAll(): AgentContextRuntime[] {
  return Array.from(this.contexts.values());
}

async reload(agentId: string, contextPath: string): Promise<AgentContextRuntime> {
  const waited = await this.waitForIdle(agentId, 30_000);
  if (!waited) throw new Error(`Reload timeout: agentId=${agentId} has in-flight turns`);
  return this.load(contextPath);
}
```

The approach is conservative:
- `AgentContextSchema` (from the compat shim) validates the FULL context.json (Core + Runtime)
- The output is cast to `AgentContextRuntime` since the schema validates all fields
- Existing code using `AgentContext` continues to work because `AgentContext = AgentContextRuntime`
- The branded `agentId` and `ownerId` flow through from the bridge schema

Note: A more aggressive approach would derive Runtime fields from conventions
(e.g., `workspacePath = /data/workspaces/${core.agentId}`), but this is deferred
to avoid changing behavior. Forge still writes full context.json with all fields.
</action>

| T-02-12 | Circular dependency — bridge imports from X9 or Forge | high | Bridge NEVER imports from consumer repos. Verified: `grep -r "from '@x9/" src/` and `grep -r "from '@forge/" src/` return zero matches. ESLint config in bridge can enforce this via `no-restricted-imports`. | V1.14 |
