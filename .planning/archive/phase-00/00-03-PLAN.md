### Task 00-03.3: Abilitare exactOptionalPropertyTypes in packages/types/tsconfig.json

**Obiettivo singolo:** abilitare la flag `exactOptionalPropertyTypes: true` in `packages/types/tsconfig.json`, fixare gli errori che emergono. Questo e il core di MGRT-02 e la prevenzione pitfall P-14.

**File toccati (assoluti):**
- `/Users/admintemp/Downloads/Claude/forge-v2/packages/types/tsconfig.json`
- File `.ts` in `packages/types/src/` e qualsiasi consumer che emerge con errori

**Step concreti:**
1. Leggere tsconfig attuale: `/Users/admintemp/Downloads/Claude/forge-v2/packages/types/tsconfig.json`.
   Contenuto verificato (linee 1-15):
   ```json
   {
     "compilerOptions": {
       "target": "ES2022",
       "module": "CommonJS",
       "moduleResolution": "node",
       "declaration": true,
       "declarationMap": true,
       "sourceMap": true,
       "outDir": "dist",
       "rootDir": "src",
       "strict": true,
       "esModuleInterop": true
     },
     "include": ["src"]
   }
   ```
2. Aggiungere `"exactOptionalPropertyTypes": true` dentro `compilerOptions`:
   ```json
   {
     "compilerOptions": {
       "target": "ES2022",
       "module": "CommonJS",
       "moduleResolution": "node",
       "declaration": true,
       "declarationMap": true,
       "sourceMap": true,
       "outDir": "dist",
       "rootDir": "src",
       "strict": true,
       "exactOptionalPropertyTypes": true,
       "esModuleInterop": true
     },
     "include": ["src"]
   }
   ```
3. `pnpm -F @forge/types typecheck` (o equivalent `pnpm -F @forge/types build --noEmit` se typecheck script non esiste) → aspettarsi errori "pattern `field?: string | undefined` non piu valid" o "passing `{ foo: undefined }` to `{ foo?: string }`".
4. Salvare log: `pnpm -F @forge/types typecheck 2>&1 | tee /tmp/forge-types-eopt-errors.log`.
5. **Triage**:
   - `0-10` errori → fix inline questo task
   - `10-50` errori → loop iterativo come 00-03.2
   - `> 50` errori → STOP, FLAG Stefano insertion point 00.1
6. Fix pattern tipici:
   - `{ foo?: string | undefined }` → `{ foo?: string }` (rimuovere `| undefined` esplicito)
   - Chiamate che passano `{ foo: undefined }` → usare spread condizionale: `{ ...(foo !== undefined && { foo }) }` o omit `foo` dalla chiamata
   - Defaults con `?? ''` dove si passava `| undefined` → usare `Partial<T>` con ommissione
7. **Ripercussione cross-service:** `pnpm -r typecheck` — `@forge/types` e importato da `services/*`. Errori che emergono nei service vanno fixati qui (parte del cascade dichiarato in 00-03 scope).
8. Loop finche `pnpm -r typecheck` exit 0 cross-repo.
9. `pnpm -r test` verde.
10. Commit atomico: `feat(forge-types): enable exactOptionalPropertyTypes + fix cascade`.

**Success criteria (binario):**
- [ ] `packages/types/tsconfig.json` contiene `"exactOptionalPropertyTypes": true`
- [ ] `pnpm -F @forge/types typecheck` exit 0
- [ ] `pnpm -r typecheck` exit 0 (cross-service)
- [ ] `pnpm -r test` exit 0
- [ ] Zero nuovi `@ts-ignore`. `@ts-expect-error` solo con TODO marker
- [ ] Error log `/tmp/forge-types-eopt-errors.log` salvato per audit

**Commit atomico:** `feat(forge-types): enable exactOptionalPropertyTypes + fix cascade`

**Rollback:** `git reset --hard HEAD~1` → restore tsconfig senza flag.

**Escalation trigger:** > 50 errori irrisolti dopo 2h → FLAG Stefano insertion point 00.1.
